IPTV All-In-One M3U + EPG Update

Current rule: keep the existing feed folder and filenames under outputs\feeds. The primary playlist remains latest_focused_playlist.m3u, and the EPG remains latest_focused_epg.xml.

Published Files

FilePurpose
outputs\feeds\latest_focused_playlist.m3uSingle TiviMate playlist containing focused live TV channels plus detected movie/show/VOD entries from the source M3U.
outputs\feeds\latest_focused_playlist.m3u8Alias copy of the same playlist for app compatibility testing.
outputs\feeds\latest_focused_epg.xmlSingle XMLTV EPG file filtered to the EPG channel IDs referenced by the published playlist when EPG rows exist.
outputs\feeds\feed_manifest.jsonMachine-readable manifest with counts, hashes, and generated file metadata.
outputs\feeds\index.htmlHuman-readable feed index for local/GitHub review.

Selection Logic

Content typeHow included
Focused live TVIncluded from data\channel_scope\focused_channels.csv.
Movies / shows / series / VODPassed through from the source M3U when detected by VOD group keywords, stream path patterns such as /movie/ or /series/, episode patterns, or VOD file extensions such as .mp4 and .mkv.
Other live TVStill excluded unless selected in the focused channel datasource.

Validation

After publishing, run scripts\validate_tivimate_feeds.py. It validates the local M3U, M3U8, EPG, URL targets, and reports detected VOD entries in the single playlist.
