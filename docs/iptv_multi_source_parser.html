IPTV Multi-Source Inventory Manager

Purpose

The manager captures IPTV provider inputs, stores each provider separately, parses playlist, XMLTV, and Xtream API data into TSV review reports, and exports source-specific filtered feeds without touching the stable SmartCDN files.

User Workflow

  1. Source: paste a provider URL or choose a saved source card.
  2. Fetch: download playlist, TV guide, and channel/movie/series metadata.
  3. Parse: build TV, movie, series, guide, group, and audit TSVs.
  4. Review: inspect groups, categories, channels, movies, shows, and filter values.
  5. Export / Publish: write filtered playlist and matching TV guide under the selected source folder.

The primary button is Run Full Inventory. It validates the selected source, fetches available data, parses inventory, exports TSV reports and workbook when supported, writes source-specific feed outputs, validates reports, and shows the latest report folder.

Source Setup

The GUI uses source cards on the left side. Active sources are shown by default; archived sources are hidden unless Show archived sources is enabled. A source card shows source name, source ID, status, live TV count, movie count, show count, and actions to use, edit, run, open reports, or archive.

Paste URL Auto-Detection

The Analyze URL action recognizes provider URLs shaped like /get.php, /xmltv.php, and /player_api.php when they include username and password query parameters. It fills:

  • Source Name and Source ID candidates derived from the host.
  • Source Type, Server, Username, Password, Playlist URL, TV Guide URL, and Xtream API URL.
  • Safe redacted URL display for documentation.

Example host behavior: cf.iptvmeezzy.shop becomes Source Name IPTV Meezzy and Source ID iptvmeezzy. The Source ID remains editable before saving.

Private Credential Handling

  • Public metadata is saved in data/source_profiles/source_profiles_public.json.
  • Credentials and raw provider URLs are saved locally in data/source_profiles/source_profiles.private.json.
  • The private file, raw provider data, generated provider reports, and source-specific web feeds are gitignored.
  • Logs and committed reports use redacted URLs. The GUI can copy private local URLs only for local use.

Fetch Workflow

Fetch All Source Data downloads playlist, TV guide, and Xtream channel/movie/series metadata when available. Individual expert actions are also available from the Tools menu and Advanced Actions panel. Every fetch writes source_fetch_manifest.tsv with redacted URL, latest path, snapshot path, bytes, hash, status, and error text.

Parse Workflow

Parse Inventory reads the latest local data for the selected source. If a provider blocks playlist export but Xtream API metadata exists, the parser can build inventory rows from Xtream API records. Parsing writes source-specific reports, scope candidates, optional workbook, and filtered feeds.

Scope and Filter Workflow

The Reports and Filters tab lets the user choose a friendly field label such as Group, Country, Language, Network, or Content Type. The value list is populated from actual parsed TSV values and shows counts beside each value. Users can search, multi-select values, preview affected records, save include/exclude rules, clear filters, and apply/export.

Rules are stored in data/channel_scope/{source_key}/scope_rules.tsv with action, field, operator, value, case sensitivity, enabled state, and notes.

Network, Country, and Language Rules

The parser does not invent network, country, or language values. It uses explicit M3U metadata, XMLTV metadata, Xtream API metadata, repo reference files for CA/US/UK/AU networks when present, and explicitly configured source defaults when such defaults exist. Unknown normalized values remain blank or are shown as unknown in the GUI, and audit TSVs record method none with confidence 0.

Troubleshooting

  • If playlist fetch is refused, fetch metadata and TV guide anyway, then parse inventory from available local data.
  • If report validation fails, run Parse Inventory again and inspect source_parse_manifest.tsv.
  • If filter values are empty, run Full Inventory first so scope_candidates.tsv exists.
  • If network matches are blank, inspect network_match_audit.tsv; blank means no repo reference matched.

Manual Workflow for a New Trial Provider

  1. Run python scripts/iptv_multi_source_gui.py.
  2. Click New Source.
  3. Paste the provider playlist, TV guide, or Xtream API URL.
  4. Click Analyze URL.
  5. Verify Source Name and Source ID, then click Save Source.
  6. Click Run Full Inventory.
  7. Open Inventory Preview and Reports and Filters to review groups, channels, movies, shows, and candidate filters.
