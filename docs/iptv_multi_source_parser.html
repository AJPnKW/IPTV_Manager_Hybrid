IPTV Multi-Source Parser Workflow

Purpose

This workflow extends the existing M3U and XMLTV tooling into a source-profile based IPTV inventory system. Each provider is fetched, parsed, reviewed, scoped, and published in its own folder so one provider cannot overwrite another provider or the existing SmartCDN feed outputs.

Source Profile Model

Public source profiles are stored in data/source_profiles/source_profiles_public.json. The public file contains only sanitized metadata: source_key, source_display_name, source_type, provider_label, timestamps, server_host, redacted URLs, credential presence, status, and notes.

Private source values are stored locally in data/source_profiles/source_profiles.private.json. That file is gitignored and is created by the GUI when credentials or raw feed URLs are saved.

Folder Structure

data/provider_feeds/{source_key}/raw/latest/
data/provider_feeds/{source_key}/raw/snapshots/{timestamp}/
reports/provider_inventory/{source_key}/latest/
reports/provider_inventory/{source_key}/snapshots/{timestamp}/
data/channel_scope/{source_key}/
web/feeds/{source_key}/

The stable SmartCDN files under web/feeds/smartcdn.* remain separate and are not overwritten by this workflow.

Private Credential Handling

  • Do not commit raw M3U URLs that include usernames, passwords, tokens, or API keys.
  • Use the GUI to save provider credentials into the gitignored private profile file.
  • Public reports and manifests use redacted URLs.
  • Local parser TSVs may contain raw stream URLs because they are local review outputs under provider-specific folders.

Fetch Workflow

  1. Open scripts/iptv_multi_source_gui.py.
  2. Add or select a source profile.
  3. Save the source so public metadata and local private values are split correctly.
  4. Use Fetch M3U, Fetch EPG, Fetch Xtream API metadata, or Fetch all.
  5. Review source_fetch_manifest.tsv in the latest report folder.

Parse Workflow

  1. Fetch source data or place local files in data/provider_feeds/{source_key}/raw/latest/.
  2. Run Parse all in the GUI or python -m backend.iptv_multi_source parse --source-key {source_key}.
  3. Review TSV outputs under reports/provider_inventory/{source_key}/latest/.
  4. Apply or edit scope rules in data/channel_scope/{source_key}/scope_rules.tsv.
  5. Parse again to refresh scope_candidates.tsv and source-specific filtered feeds.

Network, Country, and Language Derivation Rules

Network matching uses existing repo reference data from data/channel_scope/focused_channels.csv and data/channel_scope/focused_channels.json when present. The matching order is exact alias, normalized alias, then no match. If no reference match exists, network_normalized is blank, network_match_method is none, and confidence is 0.

Country and language values are populated only from deterministic source tokens, XMLTV fields, Xtream metadata, repo reference network-country values, or an explicitly configured future source default. The audit TSV records raw value, normalized value, method, and confidence.

Scoping and Filtering Model

Rules live in data/channel_scope/{source_key}/scope_rules.tsv and support include or exclude, field name, operator, value, case sensitivity, enabled state, and notes. Operators are equals, contains, starts_with, ends_with, regex, in_list, and not_equals.

Source-Specific Feed Publishing

Filtered outputs are written only to web/feeds/{source_key}/playlist.m3u, playlist.m3u8, epg.xml, index.html, and manifest.json. SmartCDN files are intentionally not touched by this source-specific parser.

Troubleshooting

  • If a fetch fails, check logs/source_fetch.log.txt and confirm the private profile has the real server, username, and password.
  • If XMLTV parsing fails, validate that data/provider_feeds/{source_key}/raw/latest/epg.xml is XMLTV and not an HTML error page.
  • If network matches are blank, review network_match_audit.tsv; blank matches mean no repo reference matched.
  • If filtered output is too broad, add exclude rules or include rules and parse again.

Manual Workflow for Adding a New Trial Provider

  1. Launch python scripts/iptv_multi_source_gui.py.
  2. Click Add source.
  3. Set Source type to xtream_codes.
  4. Enter source display name, source key, server URL, username, and password from the provider when available.
  5. Click Save source, then Show redacted URLs.
  6. Click Fetch all, then Parse all.
  7. Review the TSVs under reports/provider_inventory/{source_key}/latest/.
