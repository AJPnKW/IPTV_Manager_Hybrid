IPTV Production Pipeline Blueprint

This is the bridge between mixed input sources and TiviMate-ready M3U plus XMLTV/EPG outputs. It turns your current idea into a concrete production flow.

Core principle: the stream URL and original XMLTV source/channel identity are source facts. They should not be rewritten. Names, groups, tvg-id, labels, scope, categories, and alternate suffixes can be normalized.

Target End State

M3U

Playable Inputs

Country/category segmented playlists for TiviMate. Same channel may appear multiple times if the stream URL is different.

XMLTV

Guide Inputs

Provider schedule pulls kept as authoritative EPG sources, with normalized channel IDs where final outputs require it.

Rosetta

Mapping Layer

A persistent translation table mapping raw input labels to canonical output naming and grouping.

Review

Human Decisions

HTML review pages for scope decisions, because the final scope cannot be trusted to raw playlists alone.

Pipeline Stages

StageWhat HappensOutput
1. Source RegistryRecord every local file, internet URL, GitHub repo file, and subscribed XMLTV lineup pull as an input source.data/source_registry.json
2. Raw InventoryParse M3U entries and XMLTV channels without changing meaning. Capture source file, URL, labels, group-title, tvg-id, channel ID, lineup ID.reports/input_inventory.json
3. Scope FilterApply country/language/category rules. Default scope is CA, US, UK, AU; English; no sports; no adult; no non-English unless allowed.data/scope_decisions.json
4. Rosetta MappingTranslate raw channel names/groups into canonical names, groups, tvg-id values, and alternate numbering.data/rosetta_mappings.json
5. Alternate HandlingKeep records with different stream URLs or different XMLTV source identities. Add ALT01/ALT02 labels for same canonical channel.reports/alternates_report.html
6. Output BuildEmit segmented playlists and guide files.output/tivimate/*.m3u and output/tivimate/*.xml
7. PublishCopy production files to GitHub/HP920 hosted paths.URLs for TiviMate

Important Validation Of Your Logic

QuestionAnswer
Can the same channel appear multiple times?Yes, if the stream URL or schedule source identity is different. These are alternates/backups, not duplicates.
What is a duplicate M3U record?Same target stream URL. If URL is the same, merge metadata and keep one active output record.
What is a duplicate XMLTV schedule record?Same source lineup ID plus source channel ID. Different providers can carry similar channel names and should remain separate until mapped.
Can names/groups/tvg-id be transformed?Yes. These are presentation and matching fields. The Rosetta file exists to make those transformations repeatable.
What cannot be transformed?The actual stream URL and original provider/source identity should be preserved as source facts. Final output may reference them, but should not pretend they are different.
